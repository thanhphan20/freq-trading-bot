# PLAN-strategy-dash

## 1. Project Goal & Overview
Build a real-time Multi-Strategy Dashboard comparing 5 Freqtrade dry-run bots. 
The system focuses on aggregating Freqtrade SQLite data (`tradesv3.sqlite`), centralizing it into a unified DB, and presenting deep analytics via a modern, highly responsive Next.js frontend.

## 2. Core Features (Deep Analytics)
- **Multi-Strategy Compare**: Side-by-side metric tables (Win Rate, Trades, Avg Duration, ROI, Expected Ratio).
- **Equity Curve & Drawdown**: Time-series charts visualizing cumulative profit vs. drawdown across all 5 strategies on a single view (Recharts/Chart.js).
- **Auto Ranking System**: Real-time leaderboard evaluating strategies on Profitability, Sharpe Ratio, or custom weighted scores.
- **Daily Performance Snapshot**: Heatmap or bar chart showing daily profit/loss to easily spot consistent performers.

## 3. Architecture & Tech Stack (SQLite + Aggregator -> Next.js)
### Data Layer (Aggregator Pipeline)
- **Source**: 5 isolated `tradesv3.sqlite` databases generated by Freqtrade.
- **Aggregator**: A lightweight Python/Node.js script running on a cronjob (or background loop) that pulls live data from the 5 databases to avoid SQLite locking issues.
- **Central Data Store**: A unified `dashboard.sqlite` or PostgreSQL (if scaled) designed specifically for fast metric querying (pre-calculated equity points, daily snapshots).

### Presentation Layer (Frontend)
- **Framework**: Next.js (App Router recommended).
- **UI/UX**: Tailwind CSS for styling + Shadcn/ui for clean, modern components.
- **Data Fetching**: SWR or React Query for real-time auto-refresh features (polling the aggregated data without stress on Freqtrade).
- **Visualization**: Recharts (smooth, accessible React charts for the equity curves).
